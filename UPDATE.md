# UPDATE.md

## Purpose

Update an existing Apps Script deployment from local changes in this repository.

## Preconditions

1. Dependencies already installed.
2. `src/apps_script/.clasp.json` exists and points to the correct script.
3. You are authenticated with clasp.

## 1) Pull latest repository changes

```bash
git pull
```

## 2) Install/update local dependencies

```bash
npm install
```

## 3) Verify Apps Script binding

```bash
ls -la src/apps_script/.clasp.json
```

If missing, recreate from template and set the proper `scriptId`.

## 4) Push updated source

```bash
npm run gas:push
```

## 5) Create a new script version

```bash
npm run gas:version
```

## 6) Update deployment

In Apps Script editor:

1. `Deploy` -> `Manage deployments`
2. Edit the active deployment (keep the same deployment ID for this environment)
3. Select the newly created version number
4. Save

Important:

1. `deploymentId` is a technical ID generated by Google.
2. Do not rename/replace deployment IDs per release.
3. Keep one stable deployment ID per environment and move it forward by version.

For test deployments:

1. `Deploy` -> `Test deployments`
2. Reinstall or refresh if needed

## 7) Post-update validation

1. Open Gmail thread and confirm sidebar loads.
2. Validate auth card/token flow is still working.
3. Validate:
   - `Refresh`
   - `Create note`
   - `Create activity`
   - `Create entity` (inline and dedicated quick-create card)

User rollout behavior:

1. Existing users normally get updates automatically after propagation.
2. Users usually do not need to reinstall.
3. Re-consent can be required if OAuth scopes changed.

## Troubleshooting shortcuts

1. Re-login if push fails:
   - `npm run gas:logout`
   - `npm run gas:login`
2. Verify script binding file:
   - `src/apps_script/.clasp.json`
3. Check runbook:
   - `docs/runbooks/04-troubleshooting.md`
